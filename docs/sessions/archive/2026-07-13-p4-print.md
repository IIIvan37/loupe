## Decisions

- **La stylesheet print s'arme sur la présence de `data-print-region`** ;
  le composant n'émet l'attribut que s'il a du contenu. Contrat : une seule
  région par page — une 2e surface imprimable exigera un mécanisme de
  sélection (attribut body posé au print, portal dédié), pas un 2e attribut.
- **Arbitré — pattern draft/commit des champs numériques dupliqué** (BPM
  tempo vs mes./ligne, sémantiques différentes : Enter/Escape vs
  settle-au-blur) : pas de hook partagé tant qu'un 3e consommateur
  n'apparaît pas — même arbitrage que PopoverForm/import-menu (O.5).
- **Arbitré — `barsBadInput` dérivable en un seul état** : code déplacé tel
  quel par l'extraction, comportement sous tests — pas de churn hors sujet.
- **Arbitré — pas de footer custom** (source / page x of y / date de la
  maquette) : `page x of y` n'est pas implémentable cross-browser en CSS ;
  les en-têtes/pieds natifs du navigateur suffisent.
